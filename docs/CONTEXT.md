## 🚀 SaaS Scaling Roadmap (Next Phases — NOT YET BUILT)

> These are architectural plans. The current codebase is Campus Edition with RLS.

### Pillar 1 — Razorpay Route (Split Payments)
- Canteen onboarding API → dynamically register as linked merchant accounts
- Auto-split: 98% to canteen bank/UPI, 2% to platform wallet
- Zero manual captures, zero manual settlement steps

### Pillar 2 — Redis Pub/Sub + SSE Gateway
- Replace Supabase Realtime with in-memory Redis Pub/Sub
- `/api/realtime/sse` endpoint streams events to kitchen+student in <5ms
- Zero database read locks under peak lunch traffic

### Pillar 3 — Redis Hash Pre-Aggregation
- Maintain daily stats in Redis Hash (e.g. `stats:aditya:2026-05-25`)
- `HINCRBY` on order placed, `HINCRBYFLOAT` on revenue
- <1ms dashboard render, nightly flush cron to PostgreSQL

### Pillar 4 — Multi-Tenant Sharding (Citus)
- Schema-per-tenant for logical isolation
- Citus distributed tables sharded by `tenant_id`
- Redis routing map: tenant slug → DB connection string

### Pillar 5 — Observability (OpenTelemetry + Prometheus + Grafana)
- OTel traces on Next.js API routes → SigNoz/Jaeger
- Prometheus metrics: `payment_transaction_success_rate`, `order_processing_latency_seconds`, `redis_pubsub_active_connections`
- Grafana alerts: p99 > 500ms, webhook failure > 1%, socket drop

### Pillar 6 — Three-Tier Data Retention (Indian Compliance)
```
[ PostgreSQL Hot ] --30 days--> [ ClickHouse Warm ] --3 years--> [ S3 Cold Parquet ]
```
- Indian Income Tax Act + GST: 7-year legal retention
- ClickHouse for 12-month analytics without touching live DB

## 🐛 Known Edge Cases & Decisions

| Issue | Decision |
|-------|----------|
| Guest `user_id` is null | Ownership = possession of secure UUID v4 `orderId` |
| Bulk order creates cause micro-stutters | 50ms debounce on kitchen poll, 100ms on admin refresh |
| `expired` orders with successful payment | Both webhook + verify action revive to `placed` |
| RLS + Realtime requires anon key not service key | Student actions use anon client, admin actions use service-role client |
| iOS Safari strips referrer on Server Actions | Always bind `tenantSlug` as explicit param, never rely on referrer |
